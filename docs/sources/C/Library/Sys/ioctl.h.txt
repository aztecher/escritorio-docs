Functions
============

ioctl.h
-----------

::

    int ioctl(int fd, unsigned long request, ...)


実体がどこにあるのか追えなかった(よくわからん/MANにはあるのでそれを参照した) 
まぁとりあえず動くので良し.

ioctl関数はスペシャルファイルを構成するデバイスのパラメータを操作する. 特に, キャラクター型のスペシャルファイルの多くの動作特性を ioctl リクエストによって制御できる. 引数fdはオープンされたファイルディスクリプタでないといけない. 2番目の引数は **デバイス依存のリクエストコード** である. 3番目の引数はメモリーへの型を指定しないポインタである. ioctlの `request` には, その引数が入力パラメータと出力パラメータのどちらであるかの区別や, 3番目の引数のバイト単位のサイズといった情報がエンコードされている.

返り値は大抵の場合, 成功するとゼロが返される. ただし, ioctl() リクエスト中にはパラメータの出力に返り値を使用しているものが若干あり, その場合は成功したときに非負の値が返される. エラーの場合は -1 が返され, errnoが適切に設定される.

エラーには以下がある

* **EBADF** : fdが無効なディスクリプタである.
* **EFAULT** : argpがアクセス不可能なメモリーを参照している.
* **EINVAL** : request または argp が不正
* **ENOTTY** : fdがキャラクター型のスペシャルデバイスを参照していない.
* **ENOTTY** : 指定されたリクエストはファイルディスクリプタfdが参照する種類のオブジェクトには適用することができない.
